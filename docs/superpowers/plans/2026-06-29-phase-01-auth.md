# Phase 1 — Auth & Sessions Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Supabase email/password + OAuth (Google, Facebook) authentication with sessions, password reset, route protection for the account area, and an auth-aware header — without blocking guest checkout.

**Architecture:** Auth runs through Supabase Auth via the existing `@supabase/ssr` clients. Mutations are React 19 Server Actions that validate input with Zod, call Supabase, and return a typed `ActionResult` consumed by client forms via `useActionState`. Session refresh already happens in `src/proxy.ts`; this phase adds redirect rules there. A PKCE `/auth/callback` route exchanges email/OAuth codes for a session. The `profiles` row is auto-created by the existing `handle_new_user` trigger.

**Tech Stack:** Next.js 16 (App Router, Server Actions), React 19 (`useActionState`/`useFormStatus`), TypeScript, Tailwind v4, `@supabase/ssr`, Zod, Vitest + RTL.

## Global Constraints
- Package manager: **pnpm**.
- Auth is **not** required to browse or check out (guest checkout) — only `/account/*` is gated.
- Server-side auth uses `createClient()` from `@/lib/supabase/server`; browser uses `createClient()` from `@/lib/supabase/client`. Never use the service-role/admin client for user auth.
- Validate every form with Zod before calling Supabase; return field errors, never throw to the user.
- Design tokens only (Tailwind utilities from `globals.css`): `bg-cream`, `bg-header`, `text-ink`, `text-muted`, `bg-lime`, `bg-forest`, `text-cream`, `border-line`, `rounded-sm/md/lg/pill`, `font-display`, `font-mono`. No hardcoded hex.
- `ActionResult` shape (verbatim): `{ ok: true; redirect?: string } | { ok: false; error: string; fieldErrors?: Record<string, string[]> }`.
- Post-auth landing is `/account`; post-signout landing is `/`.
- Auth routes live under an `(auth)` route group so they do **not** inherit the storefront Header/Footer chrome.

export type ActionResult =
  | { ok: true; redirect?: string }
  | { ok: false; error: string; fieldErrors?: Record<string, string[]> };

export function fieldError(result: ActionResult | null | undefined, name: string): string | undefined {
  if (!result || result.ok) return undefined;
  return result.fieldErrors?.[name]?.[0];
}

`src/lib/auth/actions.ts`:
```ts
"use server";
import { redirect } from "next/navigation";
import { createClient } from "@/lib/supabase/server";
import { publicEnv } from "@/lib/env";
import { signInSchema, signUpSchema, emailSchema, updatePasswordSchema } from "@/lib/auth/schemas";
import type { ActionResult } from "@/lib/forms";

export async function signInAction(_prev: ActionResult | null, formData: FormData): Promise<ActionResult> {
  const parsed = signInSchema.safeParse({
    email: formData.get("email"),
    password: formData.get("password"),
  });
  if (!parsed.success) {
    return { ok: false, error: "Please fix the errors below.", fieldErrors: parsed.error.flatten().fieldErrors };
  }
  const supabase = await createClient();
  const { error } = await supabase.auth.signInWithPassword(parsed.data);
  if (error) return { ok: false, error: error.message };
  redirect("/account");
}

export async function signUpAction(_prev: ActionResult | null, formData: FormData): Promise<ActionResult> {
  const parsed = signUpSchema.safeParse({
    fullName: formData.get("fullName"),
    email: formData.get("email"),
    password: formData.get("password"),
  });
  if (!parsed.success) {
    return { ok: false, error: "Please fix the errors below.", fieldErrors: parsed.error.flatten().fieldErrors };
  }
  const ref = (formData.get("ref") as string | null) ?? undefined;
  const supabase = await createClient();
  const { error } = await supabase.auth.signUp({
    email: parsed.data.email,
    password: parsed.data.password,
    options: {
      emailRedirectTo: `${publicEnv.siteUrl}/auth/callback`,
      data: { full_name: parsed.data.fullName, ...(ref ? { referred_by_code: ref } : {}) },
    },
  });
  if (error) return { ok: false, error: error.message };
  return { ok: true };
}

export async function requestPasswordResetAction(_prev: ActionResult | null, formData: FormData): Promise<ActionResult> {
  const parsed = emailSchema.safeParse({ email: formData.get("email") });
  if (!parsed.success) {
    return { ok: false, error: "Please fix the errors below.", fieldErrors: parsed.error.flatten().fieldErrors };
  }
  const supabase = await createClient();
  const { error } = await supabase.auth.resetPasswordForEmail(parsed.data.email, {
    redirectTo: `${publicEnv.siteUrl}/auth/callback?next=/reset-password`,
  });
  if (error) return { ok: false, error: error.message };
  return { ok: true };
}

export async function updatePasswordAction(_prev: ActionResult | null, formData: FormData): Promise<ActionResult> {
  const parsed = updatePasswordSchema.safeParse({ password: formData.get("password") });
  if (!parsed.success) {
    return { ok: false, error: "Please fix the errors below.", fieldErrors: parsed.error.flatten().fieldErrors };
  }
  const supabase = await createClient();
  const { error } = await supabase.auth.updateUser({ password: parsed.data.password });
  if (error) return { ok: false, error: error.message };
  redirect("/account");
}

export async function signInWithOAuth(provider: "google" | "facebook"): Promise<ActionResult> {
  const supabase = await createClient();
  const { data, error } = await supabase.auth.signInWithOAuth({
    provider,
    options: { redirectTo: `${publicEnv.siteUrl}/auth/callback` },
  });
  if (error || !data?.url) return { ok: false, error: error?.message ?? "OAuth init failed." };
  return { ok: true, redirect: data.url };
}
```

- [ ] **Step 1: Implement callback**

`src/app/auth/callback/route.ts`:
```ts
import { NextResponse, type NextRequest } from "next/server";
import { createClient } from "@/lib/supabase/server";

export async function GET(request: NextRequest) {
  const { searchParams, origin } = new URL(request.url);
  const code = searchParams.get("code");
  const next = searchParams.get("next") ?? "/account";
  if (code) {
    const supabase = await createClient();
    const { error } = await supabase.auth.exchangeCodeForSession(code);
    if (!error) return NextResponse.redirect(`${origin}${next}`);
  }
  return NextResponse.redirect(`${origin}/login?error=auth`);
}
```

## Task 10: Route protection in proxy

**Files:** Modify `src/proxy.ts`

**Interfaces:** Consumes the existing `updateSession`. Adds: read user; if path starts `/account` and no user → redirect `/login?next=<path>`; if path is `/login` or `/signup` and user exists → redirect `/account`.

- [ ] **Step 1: Implement** — replace `src/proxy.ts` body:

`src/proxy.ts`:
```ts
import { NextResponse, type NextRequest } from "next/server";
import { createServerClient } from "@supabase/ssr";
import { publicEnv } from "@/lib/env";
import type { Database } from "@/lib/supabase/database.types";

export async function proxy(request: NextRequest) {
  let response = NextResponse.next({ request });
  const supabase = createServerClient<Database>(publicEnv.supabaseUrl, publicEnv.supabaseAnonKey, {
    cookies: {
      getAll() { return request.cookies.getAll(); },
      setAll(cookiesToSet) {
        cookiesToSet.forEach(({ name, value }) => request.cookies.set(name, value));
        response = NextResponse.next({ request });
        cookiesToSet.forEach(({ name, value, options }) => response.cookies.set(name, value, options));
      },
    },
  });

  const { data: { user } } = await supabase.auth.getUser();
  const { pathname } = request.nextUrl;

  if (pathname.startsWith("/account") && !user) {
    const url = request.nextUrl.clone();
    url.pathname = "/login";
    url.searchParams.set("next", pathname);
    return NextResponse.redirect(url);
  }
  if ((pathname === "/login" || pathname === "/signup") && user) {
    const url = request.nextUrl.clone();
    url.pathname = "/account";
    url.search = "";
    return NextResponse.redirect(url);
  }
  return response;
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)"],
};
```
> This inlines `updateSession`'s session-refresh logic so the single `getUser()` call both refreshes cookies and drives redirects. `src/lib/supabase/middleware.ts` is now unused by the proxy but kept for reference.
